## Who I Am
I am an Information Technology student specializing in software automation, data extraction, and user-centered workflow optimization. My design philosophy is rooted in **functional minimalism**: digital interfaces should actively reduce user cognitive load, mask complex database architecture, and adapt seamlessly to a user's real-world mental model. I bridge the gap between back-end technical scripting logic and clean, empathetic front-end layout execution.

### 🛠️ Tools & Proficiencies
* **UI/UX Methods:** Interaction Design, Heuristic Evaluation, Cognitive Walkthroughs, User Task Mapping, Rapid Prototyping.
* **Technologies:** Figma, Python, Visual Basic for Applications (VBA), Git.

## Case Study 1: Institutional eLearn UX Audit
### Evaluating Usability and Navigation Friction

* **Project Goals:** Investigated structural navigation barriers, fragmented user journeys, and click-heavy workflows affecting student task success rates on university portal software.
* **Methodologies:** Executed formal Heuristic Evaluations and Cognitive Walkthroughs to isolate and map interface layout flaws.
* **Core Friction Identified:** Redundant menu pathways, unnecessary cognitive load, and unoptimized visual grid layouts across standard portal endpoints.

## Case Study 2: Automated Job Tracker UI
### Designing Visual Constraints for Automation Scripts

* **System Scope:** Developed a script-driven automation system utilizing Python and VBA to parse, extract, and log job brief data directly from Outlook environments into Microsoft Excel.
* **UX Redesign Challenge:** Masking complex technical back-end execution scripts behind a clean, error-proof visual dashboard for non-technical team members.
* **Interface Implementation:** Designed an intuitive tracker input interface featuring robust visual constraints, data field validation, and clean typographic hierarchy to completely eliminate data ingestion errors.
